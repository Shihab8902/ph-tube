null and undefined in JavaScript

null and undefined both in JavaScript represents a meaningful absence of a value. While they sounds similar, they have different use cases.

null:

  • 'null' indicates the missing value of variable and object. It is often use to indicate that a variable has no value and also an object property is empty.
  • 'It is used when you want to explicitly set a variable or property to have no value.
  • For example, If you want to access an object but the object is missing 'null' will indicates that the object is missing intentionally.

undefined:

  • 'undefined' typically indicates that the variable is declared but unassigned, or an object property is not defined or missing.
  • It's is usually used as unassigned variable default value and also unassigned function default parameter.
  • It also be a default value of a return function that really does not return anything.

In practice:

  • Use 'null' when you want to indicate that a variable or object have no value.
  • Use 'undefined' when you want to indicate that a variable is declared but its property is never assigned or a property of an object is empty.

It's is worth to note that both 'undefined' and 'null' returns a falsy value upon using with conditional statement.
