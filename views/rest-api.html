REST API

A REST API is an application programming interface that indicates to the constraints of REST architectural style and allows for interaction with RESTful web services.

Key principles of a REST API include:

  1. 1. Resources: REST APIs get resources as URLs. A resource can be a piece of data or object.
  2. 2. HTTP Methods: REST APIs use HTTP methods to perform different operation on the web.
    • GET: Get the data from the server by sending http request.
    • POST: Send user declared data to the server to create a new data.
    • PUT: Send data to the server to updating an existing resource.
    • DELETE: Delete a specific data on the server.
  3. 3. Statelessness: Each request from a client to the server must contain all the information which needed to understand and process that request from the sever.
  4. 4. Uniform Interface: The interaction between the client and the server should be uniform and should be perform upon a set of rules.
  5. 5. Representation: Resources can be stored in the server in different forms like JSON, XML, HTML etc. The user can send request to get their specific data formate using headers.
  6. 6. Hypermedia as the Engine of Application State (HATEOAS): This principles suggest that the API should be give necessary links to navigate to the API

REST APIs are widely used in the web development to exchange data from one client to another. The API provides a uniform connection between different clients to send or receive data and use them.
