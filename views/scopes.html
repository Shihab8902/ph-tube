The scopes of JavaScript Variables (var, let, const)

he keywords var, let and const are used to declare variable in JavaScript, but they have different scopes and behaviors. Understanding their scopes is very important for writing maintainable, reuseable and bug-free code. Let's discuss the scope of each of these variable declaration keywords:

1. "var" scope:

The keyword 'var' have global scope and function scope upon variable declaration, and it depends where they were declared. This means the variable is accessible within the if else block as shown in the example below and also within the entire function. So, it is function scope in this regards. 2. "let" scope:

The variables with 'let' keyword has a block scope. This means the variable is belongs to the block they are defined and cannot be access outside the block, such as, within the conditional scopes, loops etc. {}.

In this example, the let y variable is accessible only within the if block where it's defined. Trying accessing the variable from outside the block will cause a reference error.

3. "const" scope:

Variables declared with 'const' keyword has similar block scope like let but the difference that the variable is immutable and cannot be modified or reassigned.

In the example, the const variable z is accessible within the function it is declared. Try the access the variable outside its scope will cause an error and also try to modify the variable also result the same.

In summary:

  • Variables declared with 'var' has function scope and global scope and also be reassigned.
  • 'let' and 'const' have block scope.
  • 'const' variables are immutable and they cannot be modified after declaration.

When choosing the keyword to use for declaration of variables in JavaScript, it is always recommended to use 'let' and 'const' over 'var'. Keyword 'var' can cause a lot of confusion and error as it can be declared globally and can be reassigned later. There is also some memory heap issue with the 'var' variables declaration.
